UX Listes sauvegardées — Comparatif

Problème signalé : l'orthophoniste voit "Aucune liste" et pense qu'elle n'a aucune liste enregistrée, alors que ça signifie "aucune liste chargée dans la sélection". Voici 3 solutions comparées.

"LISTES ENREGISTRÉES / Aucune liste" est ambigu. L'utilisatrice comprend "je n'ai aucune liste sauvegardée" au lieu de "aucune liste n'est chargée".

Option A
Reformulation minimale
On change juste le texte : "Aucune liste" → "Pas de liste active" + sous-texte avec le nombre de listes disponibles. Changement minimal, résout l'ambiguïté.

Option B Recommandée
Call-to-action explicite
Quand aucune liste n'est chargée, le bloc devient un bouton d'action clair "Charger une liste". Quand une liste est chargée, il se transforme en la carte violette habituelle. L'action est explicite, pas de confusion possible.

Option C
Séparation visuelle des concepts
Le bloc "listes" est visuellement séparé de la sélection. Il indique toujours le nombre de listes disponibles. Il agit comme un lien vers l'écran de listes, jamais comme un label d'état.
